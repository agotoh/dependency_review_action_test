# やりたいこと
RubyOnRails + Node (TypeScript),React のプロジェクトで Dependency Review Action 導入を検討している
まずは事前検証として、個人 GitHub アカウントの public リポジトリで Action の動作確認を行いたい

# 前提
## Dependency Review Action 導入の目的
PR で新規に追加される依存パッケージに既知脆弱性が含まれる場合、マージ前に自動検知してブロックする仕組みを整備する。サプライチェーン攻撃・依存汚染からの防衛線を CI に組み込むことが目的。

## 事前検証の目的
Dependency Review Action をプロジェクトのprivateパッケージに導入するには　GitHub Code Security　（$30/月) アクティブコミッターあたり
のコストがかかる
このコストに見合う効果が得られるか、実際の動作イメージをチーム全体で共有するため、個人アカウントの public リポジトリで事前検証を行う
また、代替として無料のツール（bundler-audit（Ruby/Gemfile向け）, trivy , Dependabot alerts)　の導入も検討しているので、検討のための参考情報や比較ポイントも併せて収集する
＊bundler-audit（Ruby/Gemfile向け）, trivy , Dependabot alertsも試しに入れてみられたら良い

## 検証対象のシステム構成
- RubyOnRails + Node (TypeScript),React のプロジェクト

- Actionの基本的な動作確認に注力する
